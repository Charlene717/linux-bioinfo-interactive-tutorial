CHAPTER 9 / 14

生物資訊檔案格式

FASTQ / FASTA / GTF / GFF / BED / SAM / BAM / VCF — 認識每個欄位的意義，才能用 Linux 工具操作它們。

FASTQ / FASTA / GTF / GFF / BED / SAM / BAM / VCF — knowing the columns is what lets Linux operate on them.

互動切換

點選下方格式，看欄位定義與檢查指令

FASTQ （原始 reads）

每筆 read 占 4 行：① @+read ID、② 序列、③ +（可選 ID）、④ Phred quality 字元。常壓縮為 .fastq.gz / .fq.gz。

4 lines per read: ① @+read ID, ② sequence, ③ + (optional repeated ID), ④ Phred quality characters. Almost always gzipped (.fastq.gz / .fq.gz).

@SRR12345.1 1/1
AGCTACGTACGTAGCTAGCTACGTACGT
+
IIIIIIIIIIIIIIIIIIIIIIIIIIII

常用檢查指令

zcat sample.fastq.gz | head -n 4
看第一筆 read
zcat sample.fastq.gz | awk 'END{print NR/4}'
read 總數
zcat sample.fastq.gz | awk 'NR%4==2 {print length($0)}' | sort -u
read 長度分佈

FASTA （基因組 / 蛋白序列）

交替的 >header 行 + 序列行（序列可跨多行）。

Alternating >header lines and sequence lines (sequence can span many lines).

>chr1
AGCTACGTACGTAGCT...
>chr2
TTACGTAGCTACGT...
>chrM
GATTACA...

常用檢查指令

grep "^>" genome.fa | head
列出所有 chrom
grep -c "^>" genome.fa
序列總數
samtools faidx genome.fa
建立索引
samtools faidx genome.fa chr1:1000-2000
擷取片段

GTF / GFF （gene annotation）

9 欄 tab 分隔：seqname、source、feature、start、end、score、strand、frame、attributes。1-based、closed interval（與 BED 不同！）

9 tab-separated columns: seqname, source, feature, start, end, score, strand, frame, attributes. 1-based, closed interval (different from BED!).

chr1  HAVANA  gene        11868  14409  .  +  .  gene_id "ENSG..."; gene_name "DDX11L1";
chr1  HAVANA  transcript  11868  14409  .  +  .  transcript_id "...";
chr1  HAVANA  exon        11868  12227  .  +  .  exon_number 1;

常用檢查指令

grep -v "^#" genes.gtf | cut -f3 | sort | uniq -c
feature 類型統計
awk '$3=="gene" && /protein_coding/' genes.gtf | wc -l
protein-coding 基因數
cut -f1 genes.gtf | sort -u
所有 chrom 名稱

BED （genomic intervals）

最少 3 欄：chrom、start、end。0-based、half-open（即 [start, end)），與 GTF 完全不同！可選欄位：name、score、strand、… 最多 12 欄。

Minimum 3 columns: chrom, start, end. 0-based, half-open [start, end). This differs from GTF! Optional: name, score, strand, … up to 12.

chr1  100   200   region1   .   +
chr1  500   1500  region2   .   -
chr2  1000  3000  region3   .   +
⚠️

致命差異：BED chr1 100 200 對應 GTF 的 chr1 101 200。混用座標系統是新手最大的 bug 來源——所有區段比較工具（bedtools 等）都假設 BED 是 0-based！

Critical: BED chr1 100 200 corresponds to GTF chr1 101 200. Mixing coordinate systems is a top-3 source of bioinformatics bugs — every interval tool (bedtools etc.) assumes BED is 0-based!

常用工具

sort -k1,1 -k2,2n a.bed
標準 BED 排序
bedtools intersect -a a.bed -b b.bed
兩個 BED 交集
bedtools merge -i sorted.bed
合併重疊區段

SAM / BAM （alignment 結果）

SAM 是文字版、BAM 是壓縮 binary 版（同樣資訊）。每行一筆 alignment：QNAME、FLAG、RNAME、POS、MAPQ、CIGAR、RNEXT、PNEXT、TLEN、SEQ、QUAL、… 1-based。

SAM is text, BAM is its compressed binary form (same info). Each line is one alignment: QNAME, FLAG, RNAME, POS, MAPQ, CIGAR, RNEXT, PNEXT, TLEN, SEQ, QUAL, … (1-based).

@HD  VN:1.6  SO:coordinate
@SQ  SN:chr1  LN:248956422
SRR1.1   99  chr1  10001  60  150M  =  10301  450  AGCT...  IIII...
SRR1.1  147  chr1  10301  60  150M  =  10001  -450 GCTA...  IIII...

常用工具 (samtools)

samtools view -h a.bam | head
看內容
samtools sort -o sorted.bam a.bam
排序
samtools index sorted.bam
建立索引
samtools flagstat a.bam
alignment 統計
samtools view -c -f 4 a.bam
未 mapped 的 read 數

VCF （variants）

前面是 ## meta、#CHROM header；資料 8 欄固定（CHROM、POS、ID、REF、ALT、QUAL、FILTER、INFO）+ 每個 sample 一欄 GT。1-based。

Header lines start with ## / #CHROM. Data has 8 fixed columns (CHROM, POS, ID, REF, ALT, QUAL, FILTER, INFO) + per-sample GT. 1-based.

##fileformat=VCFv4.2
#CHROM  POS    ID  REF  ALT  QUAL  FILTER  INFO            FORMAT  S1
chr1    10500  .   A    G    99    PASS    DP=42;AF=0.5    GT:DP   0/1:42
chr1    10800  .   T    C    35    LowQ    DP=12           GT:DP   1/1:12

常用工具 (bcftools)

grep -v "^#" cohort.vcf | wc -l
變異總數
bcftools view -f PASS cohort.vcf.gz
只看 PASS 變異
bcftools stats a.vcf.gz
VCF 統計
bcftools query -f '%CHROM\t%POS\t%REF\t%ALT\n' a.vcf
自訂輸出
重要對照

座標系統一張表搞定

格式起始區段範例
FASTA1-based—位置 100 = 第 100 個鹼基
GTF / GFF1-based[start, end]chr1 100 200 = 100 到 200，含兩端
BED0-based[start, end)chr1 99 200 = 同一段
SAM / BAM1-based—POS = 100 = read 開頭比到第 100 位
VCF1-based—POS = 100 = 變異發生在第 100 位
